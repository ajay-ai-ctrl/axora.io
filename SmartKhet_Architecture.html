01 — Foundation

Vision & Problem Statement

Indian farmers lose 20–40% of potential income due to lack of timely agronomic advice, poor market intelligence, and delayed disease detection. SmartKhet bridges this gap with an AI-first, offline-capable, multilingual agricultural brain accessible from a ₹5,000 Android phone.

🌾
Crop Intelligence
AI-powered recommendations based on soil type, weather, historical yield, and local market demand — personalized per farm.
🔬
Disease Detection
Computer vision model detects 50+ crop diseases from a single smartphone photo. Runs on-device with no internet required.
💬
Voice-First Interface
Speak in Hindi, Bhojpuri, Marathi, or Punjabi. SmartKhet understands and responds in the farmer's native tongue.
📊
Mandi Intelligence
Real-time and predictive mandi prices help farmers decide when to sell, where to sell, and how to hold for maximum profit.
🌦️
Hyperlocal Weather
5-day hyperlocal forecasts with advisory overlays — "rain expected Thursday, delay irrigation" type actionable nudges.
📵
Works Offline
Core features available without internet. Syncs data automatically when connectivity is restored. SMS fallback always available.

03 — Decomposition

Architectural Layers — Detailed

Each layer is independently scalable, independently deployable, and communicates via well-defined APIs and events.

📱
Layer 1
Frontend / Client Layer
  • Android App — React Native + offline SQLite cache. Edge AI via TFLite. Voice UI via on-device ASR.
  • WhatsApp Bot — Meta Cloud API webhook. Handles images, audio, and text. No app install needed.
  • IVR System — Twilio-powered voice call flow. DTMF + speech for zero-literacy users. Hindi/regional support.
  • SMS Agent — Structured SMS commands (e.g., KHET SOIL PH 6.2). Works on feature phones.
  • Progressive Web App — Lightweight browser fallback for semi-urban users.
🔀
Layer 2
API Gateway & Edge
  • Kong Gateway — Central entry point. Rate limiting, auth plugins, request routing by service.
  • JWT + OAuth2 — Phone-number based auth (OTP via MSG91). Farmer profile tokens with 30-day expiry.
  • Cloudflare CDN — Static asset serving, DDoS protection, edge caching for static advisories.
  • Request Routing — Path-based routing to microservices. /api/v1/disease → Disease Service.
  • Webhook Manager — Handles incoming WhatsApp/Twilio events and fan-out to relevant services.
⚙️
Layer 3
Backend Microservices
  • Farmer Service — Profile, land details, crop history, preferences. The source of farmer context.
  • Crop Advisory Service — Integrates soil + weather + farmer profile to generate crop/fertilizer advice.
  • Disease Detection Service — Receives image, routes to AI layer, returns diagnosis + remedy.
  • Market Intelligence Service — Aggregates mandi data, runs price prediction, generates sell recommendations.
  • Weather Service — Fetches IMD + OpenWeather data. Computes hyperlocal pin-code level forecasts.
  • NLP/Voice Service — Receives audio, performs STT, intent extraction, routes to correct service.
  • Notification Service — Manages push, WhatsApp, SMS, IVR outbound advisory notifications.
  • Analytics Service — Usage telemetry, model performance tracking, A/B testing infrastructure.
🧠
Layer 4
AI / ML Layer
  • Crop Recommender — XGBoost ensemble on soil NPK, pH, rainfall, temperature, market demand features.
  • Disease Vision Model — EfficientNet-B4 trained on PlantVillage + custom Indian crop images (50+ diseases).
  • Indic NLP Engine — IndicBERT for text understanding; OpenAI Whisper fine-tuned for Hindi/Bhojpuri/Marathi.
  • Market Predictor — LSTM + Prophet time-series model on 5-year mandi price data per commodity per region.
  • Advisory LLM — Fine-tuned LLaMA-3 or Mistral on agronomic knowledge base for conversational advice.
  • Edge Models — TFLite quantized versions of disease + crop ID for on-device inference without internet.
🗄️
Layer 5
Data Layer
  • PostgreSQL — Farmer profiles, advisory history, subscriptions. Primary transactional store.
  • MongoDB — Soil sensor readings, IoT device data, semi-structured agronomic datasets.
  • Redis — Session cache, mandi price cache (TTL 15 min), advisory cache per geo-cluster.
  • TimescaleDB — Time-series weather and soil moisture data. Hypertable partitioning by date.
  • ClickHouse — Analytical queries on usage patterns, crop yield correlations, model accuracy metrics.
  • Pinecone — Vector embeddings for semantic search of agronomic knowledge base and similar-case retrieval.
  • SQLite (Device) — On-device local store for offline operation. Syncs with PostgreSQL when online.
  • AWS S3 / GCS — Crop disease image store, trained model artifacts, bulk data exports.
☁️
Layer 6
Cloud Infrastructure
  • AWS Mumbai (ap-south-1) — Primary region. EC2 + EKS for compute. RDS Aurora PostgreSQL.
  • AWS Hyderabad (ap-south-2) — Secondary region for DR and read replicas. Low-latency for Deccan.
  • Kubernetes (EKS) — All microservices containerized. HPA for auto-scaling. Helm chart deployments.
  • Kafka (MSK) — Event streaming between microservices. Topics: soil.events, weather.events, disease.results.
  • Terraform + ArgoCD — Infrastructure as Code. GitOps-based continuous deployment pipeline.
  • Prometheus + Grafana — Full observability stack. Alerts on service latency and error rate thresholds.

06 — Flows

Step-by-Step Data Flows

Three critical user journeys mapped end-to-end, from input to advice delivery.

Flow A — Image-Based Disease Detection

📸 Farmer Photographs a Diseased Leaf

1
Capture & Pre-process
Farmer takes photo in Android app. App resizes to 224×224px, runs EXIF metadata extraction (GPS, timestamp). If offline, queues in local job queue (SQLite).
→ TFLite edge model runs immediately for instant offline result
2
Upload to API Gateway
App uploads JPEG + metadata (farmer ID, crop type, location) to POST /api/v1/disease/analyze. Kong validates JWT, rate-limits to 20 req/hour per farmer.
3
Disease Service Routing
Disease Service receives request. Stores image to S3 with UUID key. Publishes disease.analyze.requested event to Kafka topic with S3 key.
Kafka Topic: disease.events → Consumer: ML Inference Worker
4
ML Inference
Ray Serve worker pulls image from S3, runs EfficientNet-B4 inference. Returns top-3 disease predictions with confidence scores and LIME explanation heatmap.
5
Advisory Generation
Disease name → Pinecone vector search → retrieves top-5 treatment documents → LLM synthesizes personalized treatment plan in farmer's preferred language (Hindi/regional).
6
Response Delivery
Advisory returned synchronously (avg 2.3s cloud, 0.4s edge). Notification Service sends WhatsApp message and push notification. Stored in farmer's advisory history in PostgreSQL.
→ Farmer sees: Disease name + Hindi description + photo with highlighted regions + buy links for treatment
Flow B — Voice Query in Local Language

🎙️ Farmer Speaks a Question in Bhojpuri

1
Audio Capture
Farmer holds mic button, speaks query ("मेरे धान में पीलापन आ रहा है, क्या करूँ?"). App records 16kHz mono WAV, max 30 seconds. Sends to POST /api/v1/nlp/voice.
2
Speech-to-Text (STT)
NLP Service routes audio to Whisper fine-tuned on 2000+ hours of Hindi/Bhojpuri agricultural conversations. Returns transcript with language detection and confidence.
Whisper output: "mere dhan mein peelapan aa raha hai kya karun" [lang: hi-IN, conf: 0.92]
3
Intent Classification
IndicBERT classifies intent: disease.symptom.query. Entity extraction: crop=धान (rice), symptom=पीलापन (yellowing). Routes to Crop Advisory + Disease Services.
4
Context Enrichment
Farmer Service fetches profile: Ramesh, Gorakhpur UP, 2.5 acres rice farm, current stage: vegetative. Weather Service: last 5 days cloudy, high humidity. Context packaged for LLM.
5
Advisory Generation + TTS
LLaMA-3 generates response. Text-to-Speech via Google TTS (Hindi, female voice). Audio MP3 returned + text version for WhatsApp/SMS delivery.
→ "Ramesh ji, dhaan mein peelapan aksar nitrogen ki kami se hota hai. Urea 30kg per acre daaien..."
Flow C — Daily Advisory Notification

⏰ Automated 6 AM Farmer Advisory Push

1
Airflow DAG Triggered (5:30 AM IST)
Airflow scheduler triggers daily_advisory_dag. Fetches all active farmer IDs by geo-cluster (district-level batches of 10,000 farmers).
2
Data Aggregation per Cluster
Weather Service pulls 24h forecast for each district from IMD API. Mandi Service refreshes prices (Agmarknet API). Soil data from last 7 days averaged per farm cluster.
3
Personalised Advisory Generation
For each farmer: Recommendation engine scores relevance of 10 possible advice types. Top 3 selected. LLM generates 2-sentence personalised advisory in farmer's language.
Parallel processing: 10,000 advisories generated in ~4 minutes via Ray workers
4
Multi-Channel Delivery
Notification Service routes: WhatsApp (if active) → Push notification (if app installed) → SMS (if neither). IVR callback for farmers who called in the last 7 days.
07 — Resilience

Offline-First & Low-Connectivity

Designed for 2G connections (50kbps), intermittent internet, and complete offline usage. No connectivity should never mean no advice.

CONNECTIVITY TIER STRATEGY

TIER 0 — FULLY OFFLINE (No network)
  ├── TFLite disease detection runs on-device (EfficientNet 8-bit, ~12MB)
  ├── TFLite crop recommendation (XGBoost converted, ~2MB)
  ├── SQLite local DB: last 90 days of farmer advisories, mandi prices
  ├── Pre-downloaded weather forecast (synced during last online session)
  ├── Offline advisory templates: 200+ crop × disease × remedy combinations
  └── Result: 80% of core features work fully offline

TIER 1 — SMS FALLBACK (Feature phone / 0 data)
  ├── Farmer sends: "KHET SOIL 6.2 N-120 P-45 K-80" to shortcode 56070
  ├── SMS Parser Service extracts structured data
  ├── Advisory Engine generates 160-char SMS response
  ├── Response: "Wheat recommended. Apply DAP 50kg+Urea 35kg per acre. Rain Fri."
  └── Entire flow: avg 45 seconds, ₹0.15 cost per query

TIER 2 — LOW BANDWIDTH (2G/EDGE ~50kbps)
  ├── Compressed API payloads: JSON → MessagePack (60% size reduction)
  ├── Image upload: Device compresses to 100KB before upload
  ├── Progressive advisory delivery: text first, images lazy-loaded
  ├── Delta sync: only changed data synced (not full dataset)
  └── Background sync queue: user actions queued locally, sent when stronger signal

TIER 3 — NORMAL CONNECTIVITY (4G/WiFi)
  ├── Full cloud inference (better accuracy than edge models)
  ├── Real-time mandi prices, live weather
  ├── Voice queries, image analysis, LLM advisory
  └── Full sync of local SQLite with PostgreSQL

SYNC PROTOCOL: CouchDB-style conflict resolution. Device timestamp wins for soil input.
Server timestamp wins for mandi prices/weather. Advisory history merged with deduplication.

09 — Trust

Security Architecture

Farmer data is sacred. Multi-layered security, privacy-by-design, and compliance with India's PDPB (Personal Data Protection Bill).

🔐 Authentication & Authorization
Phone-number OTP via MSG91 as primary auth. JWT access tokens (15-min expiry) + refresh tokens (30-day, stored in httpOnly cookie). Role-based access: Farmer / Agri-officer / Admin. All service-to-service calls use mTLS via Istio service mesh.
🛡️ Data Encryption
All data in-transit: TLS 1.3 enforced. Data at-rest: AES-256 encryption on RDS and S3. Farmer PII (Aadhaar-linked data) stored encrypted with customer-managed KMS keys. Crop images stored with randomised UUIDs — no farmer ID in URL.
🕵️ Privacy by Design
Data minimisation — collect only what's needed for advisory. Explicit consent flows in app in local language. Right to deletion: full data purge within 72 hours of request. Anonymised data for model training — no PII in training sets. PDPB-ready architecture.
🔒 API Security
Kong rate limiting: 100 req/min per farmer. OWASP Top-10 mitigations in place. SQL injection prevention via parameterised queries only. Input validation via Pydantic schemas at every service boundary. Regular VAPT (quarterly) by third-party firm.
📡 Network Security
All services in private VPC subnets. Only API Gateway exposed via public ALB. VPN required for ops access. Security Groups: least-privilege port rules. AWS GuardDuty for threat detection. CloudTrail for full audit log of API calls.
📱 Device Security
Android app: SSL pinning to prevent MITM. Root detection — warn user on compromised device. Local SQLite database encrypted with SQLCipher. No hardcoded API keys — all secrets via AWS Secrets Manager pulled at pod startup.

11 — Revenue

Monetization Strategy

Multi-stream revenue model that keeps the core product free for farmers while monetising the ecosystem around them — inspired by Jio's playbook for rural India.

💊
AgriInput Marketplace
3–8% commission on seeds, fertilisers, pesticides sold via in-app recommendations. Input companies pay for "recommended" placement. Est. ₹1,200/farmer/year GMV.
🏦
Financial Services
Kisan Credit Card, crop insurance, agri loans via NBFC partners. 1–2% referral fee. SmartKhet's risk data (crop health, yield history) enables better underwriting.
🏢
B2G / FPO SaaS
White-label SmartKhet for state agriculture departments, Farmer Producer Organisations. ₹50–200/farmer/year SaaS licensing. High-margin, gov. procurement.
📡
Data Intelligence
Anonymised crop health, soil, and yield datasets sold to commodity traders, insurance companies, and climate research orgs. ₹50L–2Cr/year data licensing.
